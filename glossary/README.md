# Glossary

Every concept the problem write-ups lean on is explained once here, in plain
language with no assumed programming background, and linked from wherever it
appears. Use it as a study path: follow a term, then follow *its* related terms
only as deep as you need.

| Term | In one line |
|---|---|
| [Big-O Notation](big-o-notation.md) | Shorthand for how much slower (or more memory-hungry) a method gets as the input grows. |
| [Hash Map](hash-map.md) | A container that stores key → value pairs and can find any key almost instantly. |
